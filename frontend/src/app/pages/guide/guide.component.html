Global interface overview
@if (selectedCategory == 'global-interface') {
Global interface overview The Heatlink app is a dashboard that allows you to interact with your Heatzy smart heating modules. Its interface is fairly straightforward:

- In the top bar, you can find the current time and date on the left, and the current weather on the right. These auto update periodically, so no need to refresh your page! The weather widget only appears if you have set the LATITUDE and LONGITUDE environment variables in the app.
- Under the main title, you can find the list of devices linked to your Heatzy account. For more information on managing devices, see the other sections.
- In the bottom bar, on the left, you can find the language picker. It currently allows you to switch between French and English. All units are also affected.
}
Managing target temperatures
@if (selectedCategory == 'target-temperatures') {
Managing target temperatures On each individual device, you can set the target temperatures for Comfort and Eco modes. Clicking the highlighted value (in degrees Celsius, see on the image) will open a modal that will prompt you to choose the target temperature.
 Once you press "Confirm", it'll immediately be applied to your device.
}
Device heating status
@if (selectedCategory == 'heating-status') {
Device heating status When your device is currently heating, you'll notice a small orange blinking dot next to the device's name.
This information is live: no need to refresh the page to see it appear or disappear. See the Data Synchronization section for more information.
}
Changing the manual heating mode
@if (selectedCategory == 'manual-heating-modes') {
Changing the manual heating mode For each individual device, you can change the current heating mode manually. Note that if a special mode such as motion detection, vacancy or boost is enabled, you cannot change it manually.
If the timer is on (in the heating schedule section), changing the mode manually will only do so until the next scheduled mode change.
From left to right:

- Off: the heater is turned off.
- Comfort: the heater will turn on until it reaches the specified target comfort temperature.
- Eco: the heater will turn on until it reaches the specified target eco temperature.
- Frost protection: the heater will turn on only if the temperature drops below 7 degrees Celsius.

The extra button on the right allows you to manage special modes, heating schedules and data histories (see other sections for more information).
}
Using comfort-1 and comfort-2 modes
@if (selectedCategory == 'comfort-1-2') {
Using comfort-1 and comfort-2 modes You can set your heating mode to comfort-1 or comfort-2. Those modes respectively set your comfort target temperature to your specified target minus one or two degrees celsius.
To enable these modes, you can click the "comfort" manual mode while it is already selected. It will open a context menu, letting you choose between the two options.
}
Updating the heating schedule
@if (selectedCategory == 'heating-schedule') {
Updating the heating schedule To open the heating schedule utility, you can press the "+" button on the right hand side of a device card (next to the manual heating modes), and select the "Heating schedule" item. It will open another card under the selected device. On a computer:
To select a time range for which to change the scheduled heating mode, simply click and drag over the time period you want to select. A small popup will appear next to your cursor, allowing you to choose a specific mode for the selected time frame. Note that time frames can only be selected in increments of thirty minutes.

On a mobile device:
To select a time range for which to change the scheduled heating mode, simply click on the start of the requested time frame, and a second time on the end of the time frame. A small popup will appear, allowing you to choose a specific mode for the selected time frame. Note that time frames can only be selected in increments of thirty minutes.
Once you are satisfied with your schedule, you can press the "Save schedule" button on the bottom right hand corner to save your changes.
}
Changing the timer mode
@if (selectedCategory == 'timer-mode') {
Changing the timer mode Even if you have set a schedule for your device, it might not be active. You can enable or disable the timer mode (or scheduling mode) at will using the scheduling utility. Simply press the corresponding icon (left is off, right is on) above the schedule. No need to save, changing the timer mode is instantaneous.
}
Duplicating a day's schedule
@if (selectedCategory == 'duplicate-day') {
Duplicating a day's schedule Remaking the same schedule for all seven days of the week might get a little bit tedious. Thankfully, there is a built-in utility to duplicate a day's schedule to other days of the same schedule. Simply click on the day's label and select the "Copy to other days" option.
 A modal will open, allowing you to select which days you wish to copy the schedule to. Pressing confirm will update the schedule accordingly. Make sure to save your changes!
}
Using schedule presets
@if (selectedCategory == 'using-presets') {
Using schedule presets Let's say you have a different heating schedule in winter and in summer (which would make sense!). Surely, you wouldn't want to change the schedule manually every time the seasons change, right?
Thankfully, there is a built-in utility that allows you to save and reuse scheduling presets.
You access those tools by pressing the three dots on the right hand side of the title in the heating schedule utility. Pressing the "Save as preset" button will open a modal, allowing you to input a name (mandatory) and a description (optional) for your preset. Once you press confirm, it'll be saved in the database and reusable in the future. Pressing the "Load preset" button will open a modal, allowing you to choose from any previously saved schedule. Once you click one, it'll automatically appear on the schedule for the selected device. Make sure to save your changes!
You can also delete a preset in the "Load preset" modal, by clicking the delete icon on the top right hand side of each preset.
}
Duplicating a schedule to other devices
@if (selectedCategory == 'duplicating-to-devices') {
Duplicating a schedule to other devices Once you've carefully created your schedule for one device, chances are you'll want to use the same schedule for other devices in your home. Thankfully (once again), there are built-in tools in Heatlink that allow you to do just that.
In the three dots on the right hand side of the title in the heating schedule utility, you can select the "Duplicate to devices" item. A modal will open, allowing you to select which devices you wish to duplicate the schedule to. Simply press confirm for the changes to be saved.
}
Using motion detection
@if (selectedCategory == 'motion-detection') {
Using motion detection You can enable motion detection mode by clicking the "+" button on the right hand side of each device card, next to the manual heating modes. You can then select the "Motion detection" item, which will turn it on for the selected device.
In motion detection mode, the device auto sets itself to comfort mode. After thirty minutes of inactivity, it'll switch to comfort-1 mode. After sixty minutes, it'll switch to comfort-2. After ninety minutes, it'll switch to eco mode.
When motion detection mode is active, the "off" button is replaced by a "Motion" label in the heating modes bar. You can turn off motion detection mode by clicking the "Motion" button that replaced the "off" button in the heating modes bar. A prompt will ask you if you are sure you wish to disable it: press "Confirm" to disable it.
}
Using vacancy mode
@if (selectedCategory == 'vacancy') {
Using vacancy mode You can enable vacancy mode by clicking the "+" button on the right hand side of each device card, next to the manual heating modes. You can then select the "Vacancy" item, which will open a modal to let you choose the amount of time for which you will be away.
In vacancy mode, the heater is automatically set to frost protection mode for the amount of time you set.
When vacancy mode is active, the "off" button is replaced by a "Vacancy" label in the heating modes bar. In the modal that opens when you select the "Vacancy" item, you can click the date that is displayed in the middle to choose the date at which you will be back. Heatlink automatically calculates the number of days between today and the target date, and sets the device to vacancy mode for that period of time as soon as you press "Confirm". You can turn off vacancy mode by clicking the "Vacancy" button that replaced the "off" button in the heating modes bar. A prompt will ask you if you are sure you wish to disable it: press "Confirm" to disable it.
}
Using boost mode
@if (selectedCategory == 'boost') {
Using boost mode You can enable boost mode by clicking the "+" button on the right hand side of each device card, next to the manual heating modes. You can then select the "Boost" item, which will open a modal to let you choose the amount of time for which the device will be in boost mode.
In boost mode, the heater is automatically set to comfort mode for the amount of time you set. This mode takes priority over the timer.
When boost mode is active, the "off" button is replaced by a "Boost" label in the heating modes bar. In the modal that opens when you select the "Boost" item, you can set the amount of time the device will be in boost mode for. This can be set in thirty minute increments, and can range from 30 minutes to 2 hours. You can turn off boost mode by clicking the "Boost" button that replaced the "off" button in the heating modes bar. A prompt will ask you if you are sure you wish to disable it: press "Confirm" to disable it.
}
Using lock mode
@if (selectedCategory == 'lock') {
Using lock mode You can enable lock mode by clicking the "+" button on the right hand side of each device card, next to the manual heating modes. You can then select the "Lock" item, which will instantly set the device to locked mode.
Lock mode doesn't impact the way the device behaves in the application. It merely locks the physical interface of the device, to prevent children from touching it, for instance.
When lock mode is active, a padlock appears on the left hand side of the device name. If you wish to disable lock mode, you can re-open the "+" menu, and click the same button you used to enable lock mode (which should now be labeled "Unlock"). Upon clicking it, the device will instantly be unlocked, and the padlock will disappear.
}
Viewing histories
@if (selectedCategory == 'history') {
Viewing histories You can view the data history for each of your devices by pressing the "+" button on the right hand side of each device card, next to the manual heating modes. You can then select the "History" item, which will open an extra widget below the selected device.
 Two graphs will be available: temperature history, and humidity history.
If you've set the LATITUDE and LONGITUDE environment variables, you will also be able to compare your Heatlink data with external temperature and humidity, thanks to the OpenMeteo API.
Hovering over the graphs will give you the exact value of each data point.
Note that Heatlink does data archival on its own: the server needs to be running at all times to keep a consistent history of your devices' temperature and humidity values. You can set the time range for the history from a list of presets, available on the top left corner of the widget.
Note that the data is auto-refreshed: no need to click again or refresh the page, as it fetches recent history every minute from the server.
}
Changing the language
@if (selectedCategory == 'language') {
Changing the language You can change the app's language by clicking the current language label on the bottom left hand corner of the screen. A context menu will open, allowing you to choose between either English or French. The app will instantly be translated into the target language without needing a refresh.
}
Data synchronization
@if (selectedCategory == 'data-sync') {
Data synchronization All the data in Heatlink is connected live to the server (that includes room temperature, humidity, heating status...). That means if you change the heating mode on your phone's Heatzy app, for instance, it'll automatically be updated on the Heatlink dashboard. As a rule of thumb, if you need to refresh the page to see data change, that's considered a bug: don't hesitate to let me know!
}
